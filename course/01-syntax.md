# Модуль 1. Синтаксис и основы Go 🟢

**Срок:** 3–4 недели  
**Цель модуля:** выучить базовый синтаксис Go и понять идиоматику языка. На этом фундаменте строится всё остальное.

---

## Урок 1.1. Установка и `go mod`

**🎯 Цель:** понять модульную систему.

**📚 Теория:**
- Установка, версии Go (`go version`)
- - Структура проекта, `go.mod`, `go.sum`
  - - `GOPATH` vs модули, vendoring
    - - Команды: `go run`, `go build`, `go test`, `go mod init`, `go mod tidy`, `go get`
     
      - **💻 Практика:** создай модуль `hello`, напиши Hello World, подключи стороннюю библиотеку (например `github.com/fatih/color`).
     
      - ---

      ## Урок 1.2. Типы данных

      **📚 Теория:** `int`/`int64`/`uint`, `float32`/`float64`, `bool`, `string`, `byte`, `rune`, явные приведения типов, значения по умолчанию (zero values).

      **💻 Практика:** напиши конвертер валют с парсингом строки в `float64` через `strconv`.

      **✅ Чек:** разница между `byte` и `rune`, почему `int` разный размер на 32 и 64-битных платформах.

      ---

      ## Урок 1.3. Переменные и константы

      **📚 Теория:** `var`, `:=`, блочные объявления, `const`, типизированные вс нетипизированные константы, `iota` и его трюки (битовые флаги).

      **💻 Практика:** объяви enum для статусов задачи (`StatusTodo`, `StatusDone`, ...) через `iota`.

      ---

      ## Урок 1.4. Управляющие конструкции

      **📚 Теория:** `if` с init-выражением, единственный цикл `for` (3 формы + `for range`), `switch` и type switch, `break`, `continue`, метки циклов (labels).

      **💻 Практика:** FizzBuzz через `switch` без `if`.

      ---

      ## Урок 1.5. Функции

      **📚 Теория:** объявление, множественный возврат, именованные возвраты, variadic-функции, замыкания, `defer` (LIFO-порядок), `panic`/`recover`.

      **💻 Практика:** напиши функцию `withTiming`, которая замеряет время выполнения через `defer`.

      **✅ Чек:** почему `defer file.Close()` это идиома.

      ---

      ## Урок 1.6. Указатели

      **📚 Теория:** `&` и `*`, передача по значению vs по указателю, `nil`, отсутствие арифметики указателей.

      **✅ Чек:** когда передавать в функцию указатель, а когда значение.

      ---

      ## Урок 1.7. Слайсы

      **📚 Теория:** внутреннее устройство (ptr/len/cap), `make`, `append`, разрезание (slicing), ловушки (модификация одного слайса меняет другой).

      **💻 Практика:** реализуй `Filter[T]`, `Map[T,U]`, `Reduce[T,U]` без дженериков (через `[]int`).

      **✅ Чек:** почему `append` иногда возвращает новый слайс; что такое рост cap.

      ---

      ## Урок 1.8. Мапы

      **📚 Теория:** создание, проверка наличия ключа (`v, ok := m[k]`), удаление (`delete`), итерирование (порядок не гарантирован), nil map vs пустая.

      **✅ Чек:** разница nil map и пустой map.

      ---

      ## Урок 1.9. Строки и руны

      **📚 Теория:** неизменяемость, UTF-8, `[]byte` vs `[]rune`, `strings`, `unicode/utf8`, `strings.Builder`.

      **💻 Практика:** напиши функцию реверса строки с кириллицей правильно (через руны).

      ---

      ## Урок 1.10. Структуры и методы

      **📚 Теория:** объявление `struct`, теги полей, value receiver vs pointer receiver, встраивание (эмбеддинг).

      **💻 Практика:** структура `Task` с методами `MarkDone`, `String`, `IsOverdue`.

      **✅ Чек:** когда value receiver, а когда pointer receiver.

      ---

      ## Урок 1.11. Ошибки

      **📚 Теория:** интерфейс `error`, `errors.New`, `fmt.Errorf` с `%w`, `errors.Is`, `errors.As`, свои типы ошибок, sentinel errors.

      **💻 Практика:** своя ошибка `ErrTaskNotFound`, оборачивание через `fmt.Errorf("load: %w", err)`, проверка через `errors.Is`.

      ---

      ## 🎯 Проект модуля: CLI Todo

      Напиши CLI-утилиту `todo` с сохранением в JSON.

      Подкоманды:
      - `todo add "Купить хлеб"` — добавить задачу
      - - `todo list` — показать список
        - - `todo done <id>` — отметить выполненной
          - - `todo remove <id>` — удалить
           
            - **Критерии приёмки:** проект компилируется, `golangci-lint run` чист, данные персистятся между запусками, ошибки обработаны через `errors.Is`/`errors.As`.
